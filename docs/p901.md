# P901 Smart ID Card GPS Walkie-Talkie

## Overview

The P901 is an ID-card-form-factor device combining GPS tracking, GPRS data upload, SOS, punch-in/out, and walkie-talkie intercom. It is a **passive SMS-only device** — all configuration and querying is done by sending SMS commands to the device's SIM number; the device replies by SMS.

> **No TCP/IP, REST, or socket support.** All communication is SMS-based. The `tcp-listener` service is not used for P901.

**Driver class:** `App\Drivers\P901Driver`
**Communication mode:** `passive` (we send SMS → device replies by SMS)
**Slug:** `p901`

## Default Password

All SMS commands include the device password. The default password is `123456`. It is stored per-device in `devices.metadata.password` and must be used in every command.

**Command format:** `<command><password> <parameters>`

## GPS Tracking Modes

| Mode | SMS command | Behaviour |
|---|---|---|
| 0 — Power saving | `md123456 0` | Device does not report actively; query manually |
| 1 — Timing report | `md123456 1 20M` | Reports every N minutes (e.g. every 20 min) |
| 2 — Real-time | `md123456 2 60S` | Continuous reporting; minimum interval ~10 s |
| 3 — Smart (default) | `md123456 3 30S` | Reports on vibration; sleeps when idle (heartbeat every 5 min) |

## Incoming SMS Reply Format

The P901 sends SMS replies in response to query commands. Known reply patterns:

### Location Reply (from `query123456`)
```
Lat:31.549700,Lon:74.343600,Speed:0km/h,Course:0,T:2026-05-06 10:30:00,Battery:75%,GPS:Fixed
```
Fields: `Lat`, `Lon`, `Speed` (km/h), `Course` (degrees), `T` (UTC datetime), `Battery` (%), `GPS` (Fixed/No Fix)

### Status / Check Reply (from `check123456`)
```
IMEI:123456789012345,Ver:1.2.3,Signal:18,Battery:80%,Mode:3,IP:yourdomain.com,Port:7018
```
Fields: `IMEI`, `Ver` (firmware), `Signal` (0–31 CSQ), `Battery` (%), `Mode` (0–3), `IP`, `Port`

### Low Battery Alert (device-initiated)
```
Low battery alarm! IMEI:123456789012345,Battery:18%,Lat:31.549700,Lon:74.343600
```

### Punch-In / Punch-Out (device-initiated)
```
Punch in! IMEI:123456789012345,T:2026-05-06 08:00:00,Lat:31.549700,Lon:74.343600,Battery:90%
Punch out! IMEI:123456789012345,T:2026-05-06 17:00:00,Lat:31.549700,Lon:74.343600,Battery:45%
```

### SOS Alert (device-initiated)
```
SOS! IMEI:123456789012345,T:2026-05-06 12:00:00,Lat:31.549700,Lon:74.343600,Battery:60%
```

## Normalized Payload Fields

After `P901Driver::parseIncoming()` processes a raw SMS, it returns a `NormalizedPayload` with:

| Field | Type | Source |
|---|---|---|
| `imei` | string | From SMS body or device lookup |
| `latitude` | float\|null | Parsed from `Lat:` |
| `longitude` | float\|null | Parsed from `Lon:` |
| `speed` | float\|null | Parsed from `Speed:` |
| `course` | int\|null | Parsed from `Course:` |
| `battery` | int\|null | Parsed from `Battery:` (strip %) |
| `signal_strength` | int\|null | Parsed from `Signal:` (CSQ 0–31, normalize to 0–100%) |
| `gps_fixed` | bool\|null | `GPS:Fixed` → true |
| `event_type` | string\|null | `PunchIn`, `PunchOut`, `SosPressed`, `LowBattery`, `LocationUpdated` |
| `recorded_at` | datetime\|null | Parsed from `T:` field |
| `raw` | string | Original SMS body |

## Implementation Notes for Driver

- Password is read from `device->metadata['password'] ?? '123456'`
- All commands must be built as pure strings — no side effects in `buildCommand()`
- `canParseIncoming()` returns `true` if the SMS contains `Lat:` or known P901 keyword patterns (`Punch in!`, `Punch out!`, `SOS!`, `Low battery alarm!`, `IMEI:`)
- CSQ signal (0–31) maps to percentage: `round(($csq / 31) * 100)`
- If `GPS:No Fix`, set `latitude/longitude = null` and `gps_fixed = false`
- `parseIncoming()` must never throw — return partial payload with available fields
